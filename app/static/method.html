How the trust score works

← Trust Desk

No ground truth exists for these 10,088 facilities. Instead of pretending certainty, Trust Desk scores how well each claim is supported by the record itself — and says explicitly what it cannot know. Every number below is computed in SQL over Unity Catalog tables; nothing is a black box.

The score: 0–100 from four observable signals

SignalPointsWhat it measures
Corroboration40For each claimed capability (ICU, surgery, …): does the same signal appear independently in equipment, procedures, or the description? 2+ fields = full credit, 1 field = half, claim-only = zero. This is the discriminating signal: 45–51% of high-acuity claims in this dataset have zero corroboration.
Completeness208 checks (plausible bed count, doctor count, year, real description, specialties, non-filler equipment & procedures). Separates curated records from thin scrapes.
Web presence25Verifiable footprint: official website (directories like justdial excluded), email, phone, Facebook, active social metrics.
Geo validity15Coordinates inside India, postal code exists in the 165,627-row India Post directory, state consistent with that postal record.

Penalties for suspicious data

  • Junk/misaligned row (−15) · "multi-speciality" with <10 beds (−5)
  • Absurd values: capacity 0 or >5,000 · >1,000 doctors · bogus founding year (−3 each)
  • Surgery claim with no anesthesia signal at a small facility (−3) — kept soft because 73% of all surgery claims lack an anesthesia mention, including major real hospitals

Tiers

≥75 Trusted 50–74 Plausible 25–49 Thin evidence <25 Red flag

Uncertainty: absence of data ≠ absence of care

A facility with no equipment list cannot be corroborated by equipment — that is missing data, not a failed check. For every facility we also compute the score it could reach if its unobservable fields were favorable. The width of that range is the facility's data deficit: ≤10 solid ≤25 moderate >25 speculative. 1,284 of 10,088 facilities are moderate or speculative — the app shows the range, not just the point.

Three layers of skepticism

  • SQL evidence score — deterministic, explainable, covers all 10,088 rows.
  • AI evidence audit — on-demand LLM cross-examination (Databricks FM API): verdict, confidence, exact quotes only, medical-consistency checks (ICU⇒ventilators, surgery⇒anesthesia). Cached in Lakebase, every reasoning step traced in MLflow. It can and does disagree with the SQL score.
  • Human override — the planner's judgment wins: confirm / dispute / flag with a note, persisted across sessions, displayed on every ranking.

Does the signal mean anything? (self-validation without ground truth)

If corroboration were random keyword noise, whether a facility's equipment list supports a claim would be independent of whether its procedures list does. It isn't. Among facilities where both fields are observable:

Corroboration clusters on the same facilities far above chance — consistent with a latent "actually has the capability" factor rather than keyword coincidence. This is the strongest validation available when no ground truth exists; field visits (the override layer) remain the only true test.

Known limits (read before trusting the tool that measures trust)

  • Corroboration uses keyword patterns with word boundaries; paraphrases can be missed. The AI audit exists precisely to catch these.
  • All fields were extracted from the same web sources by an LLM pipeline — "independent fields" are independent extractions, not independent sources. True independence needs site visits: that is what the override layer records.
  • Web presence rewards digitally-visible facilities; rural facilities may be under-scored. The uncertainty range partially compensates (missing web data widens it).
  • The score ranks evidence quality, not clinical quality.
